# Translation Toolkit

Набор Python-утилит для автоматизации переводов YAML-файлов. Основная цель — минимизировать расход контекстного окна при переводе через Claude Code, разбивая большие файлы на чанки.

## Установка

```bash
pip install -r Tools/requirements.txt
```

Единственная зависимость — `ruamel.yaml` (сохраняет стиль кавычек, порядок ключей и комментарии при редактировании YAML).

## Утилиты

### extractor.py — извлечение одного языка

Забирает из YAML-файла только строки на указанном языке, сохраняя структуру. Используется для подготовки файла к переводу — убирает все языки кроме исходного.

```bash
python Tools/extractor.py <файл.yaml> [язык]
```

- `язык` — код языка для извлечения (по умолчанию `en`)
- Результат сохраняется в `./tmp/<имя_файла>/<язык>.yaml`

**Пример:**

```bash
python Tools/extractor.py server.yaml en
# → ./tmp/server/en.yaml
```

Было (server.yaml):
```yaml
Sections:
  Server:
    Key1:
      en: "English text"
      ru: "Русский текст"
```

Стало (tmp/server/en.yaml):
```yaml
Sections:
  Server:
    Key1:
      en: "English text"
```

---

### splitter.py — разбиение на чанки

Разбивает YAML-файл на несколько файлов заданного размера. Каждый чанк — валидный YAML с полной иерархией `Sections > SectionName > keys`. Разбиение идёт по ключам, структура не ломается.

```bash
python Tools/splitter.py <файл.yaml> [размер_КБ]
```

- `размер_КБ` — максимальный размер чанка в килобайтах (по умолчанию `20`)
- Результат: `./tmp/<имя_файла>/<язык>/1.yaml, 2.yaml, ...`

**Пример:**

```bash
python Tools/splitter.py ./tmp/server/en.yaml 20
# → ./tmp/server/en/1.yaml, ./tmp/server/en/2.yaml, ...
```

---

### inserter.py — вставка переводов в файл

Берёт переведённые чанки (каталог или один файл) и вставляет переводы в целевой YAML-файл. Работает в двух режимах:

- **Если целевой файл существует** — мержит переводы в него (добавляет/обновляет язык)
- **Если целевой файл не существует** — создаёт новый файл только с переводами

```bash
python Tools/inserter.py <каталог_или_файл> <целевой.yaml>
```

**Пример 1 — вставка в существующий файл:**

```bash
python Tools/inserter.py ./tmp/server/it/ server.yaml
# → server.yaml обновлён
```

Было (server.yaml):
```yaml
Sections:
  Server:
    Key1:
      en: "English text"
      ru: "Русский текст"
```

Стало:
```yaml
Sections:
  Server:
    Key1:
      en: "English text"
      ru: "Русский текст"
      it: "Testo italiano"
```

**Пример 2 — создание отдельного файла переводов:**

```bash
python Tools/inserter.py ./tmp/ios/it/ ios-it.yaml
# → ios-it.yaml создан
```

Результат (ios-it.yaml):
```yaml
Sections:
  iOS:
    Key1:
      it: "Testo italiano"
    Key2:
      it: "Un altro testo"
```

### validator.py — валидация результата

Проверяет YAML-файл на корректность структуры и консистентность переменных в переводах. Выводит ошибки с номерами строк.

```bash
python Tools/validator.py <файл.yaml> [целевой_язык]
```

- Если указан `целевой_язык` — проверяет его наличие для каждого ключа и сравнивает переменные с `en`
- Exit code: `0` — ошибок нет, `1` — есть ошибки

**Что проверяется:**

| Проверка | Описание |
|----------|----------|
| YAML-структура | Файл парсится, соблюдена иерархия Sections > Section > Key > Lang |
| Наличие языка | Для каждого ключа с `en` должен быть целевой язык |
| Переменные `{var}` | Набор переменных в переводе совпадает с `en` |
| Переменные `%N$type` | Для мобильных файлов — printf-плейсхолдеры совпадают с `en` |
| Вложенность | Язык находится на правильном уровне (не сместился) |

**Пример вывода:**

```
ERROR line 4306: Key 'Server.NeedMorePhotos' — variable mismatch in 'cs': missing {'{photos-count}'}, extra {'{počet foto}'}
WARN  line 16: Key 'B2B.SendDescription' — language 'it' not found
OK: b2bfrontend.yaml [it]
```

## Использование через Claude Code

Достаточно написать промпт на естественном языке. Поддерживаются два воркфлоу:

### Воркфлоу 1 — добавить язык в существующий файл

```
добавь итальянский язык в файл server.yaml
```

Claude Code выполнит: extract → split → translate → **insert в оригинал** → validate

### Воркфлоу 2 — создать отдельный файл переводов

```
создай файл итальянских переводов на основе ios.yaml
```

Claude Code выполнит: extract → split → translate → **создать ios-it.yaml** → validate

Разница только в шаге 4: `inserter.py` получает либо существующий файл (мерж), либо новый (создание).

### Общий пайплайн

1. **Извлечение** — `extractor.py` вытащит английские строки
2. **Разбиение** — `splitter.py` разобьёт на чанки по 20КБ
3. **Перевод** — Claude Code прочитает каждый чанк, переведёт и запишет результат
4. **Сборка** — `inserter.py` вставит переводы в оригинал или создаст новый файл
5. **Валидация** — `validator.py` проверит структуру и переменные
6. **Исправление** — если валидатор нашёл ошибки, Claude Code их исправит

Промежуточные файлы создаются в каталоге `./tmp/` (добавлен в `.gitignore`).
